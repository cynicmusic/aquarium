# Cuttlefish anatomy review — v004 / sheet v003

## Score: 4 / 10

Silhouette reads as "some ovoid sea-creature with a bundle of legs," not
specifically as *Sepia*. Every one of the six anatomical cues lands on the
wrong side of the comparison.

## 1. Mantle silhouette — fails

Refs show a **flattened dorso-ventral shield** with a prominent dorsal hump
~1/3 back from the nose and a visible mantle collar. `buildMantle` produces
an almost axially symmetric capsule: `mantleHeight = 0.28` vs
`mantleRadius = 0.35` (only 20% flatter), with a tiny `+0.12 * exp(...)`
hump at `t = 0.28`. The master reference is at least 2:1 flat. In v004
the mesh looks like a **loaf of bread**. Belly squash (`cy *= 0.75`) is
fine; the dorsal is the problem.

## 2. Arms as a crown — fails

Refs show 8 arms forming a **tight rose in front of the mouth**. `armSpine`
distributes them as `spreadY = (local/3 - 0.5) * 0.35`, `spreadZ = sideSign
* (0.12 + local*0.04)` — a flat **2×4 rake** hanging under the mantle like
dangling noodles. Only ~4 are distinguishable; the rest stack behind.
Additionally, `forward = -0.6` throws arms straight forward rather than
forward-and-inward, so tips diverge instead of converging toward the
"praying hands" resting pose.

## 3. Hunting tentacles — fails (read as more arms)

Club exists (good), but visually indistinguishable:

- `tentacleLength = 0.95` vs `armLength = 0.55` is only **1.7×**. Refs show
  2.5–3× when partially extended.
- `tentacleBaseRadius = 0.030` vs `armBaseRadius = 0.055` is only 1.8×
  thinner; should be whip-like.
- `tentacleClubRadius = 0.040` is barely larger than arm base (0.055), so
  the club doesn't pop.
- Retracted tentacles sit **outside** the arm crown (`start.z = sideSign *
  0.045`). Real cuttlefish hide them in pockets between arms III and IV.
  No pocketing here; tentacles are always visible.

## 4. Side fins — partial, reads wrong

`buildSideFin` runs t=0.05→0.95 along the flank with animated ripple — OK.
Problems:

- `finWidth = 0.10` vs `mantleRadius = 0.35` → only 29% flare; refs show
  40–50%. The `finMask = sin(...)` pinches further at the ends.
- Fin is a **flat horizontal plane** (all y=0 in the generator). Real fins
  attach near the dorsal-ventral midline and have rest-curvature — from the
  side, the current fin disappears into a line.
- No attachment fairing: `zOffset = side * (R * profileScale + finW * s)`
  sticks out like a paper cut against the mantle surface.

## 5. Head / mantle junction — fails (most recognisable feature missing)

Refs show a distinct collar where mantle opening meets head, with eyes
clearly on the **head**, not the mantle. In code:

- `headSize` / `headOffset` exist in DEFAULTS but there is **no
  `buildHead` function** — the "head" is just the rounded front of the
  mantle (`t < 0.12` sine nose). No visible mantle collar.
- `eyeOffsetX = -0.55` puts eyes at 55% back along the mantle — laterally
  placed (good) but mounted on the **shoulders of the loaf** rather than a
  distinct head.
- No funnel/siphon on the ventral side of the mantle opening.

## 6. 4×4 variant sheet — subtle, not distinguishable

Rows (squat/round → elongated) differ by maybe 15% body length on screen —
reads as the same creature at different zooms. mantleLength range 1.2→2.0
is too narrow; radius/height aren't varied in sympathy. Columns (calm →
striking) all show tentacles already out; only the far-right column has
noticeably longer tentacles. Arms don't re-pose between calm and hunting.
The grid shows **subtle parametric noise around one anatomy**, not four
body types × four behaviours.

## Three specific geometry fixes

### Fix 1 — `buildMantle`: real cuttlefish cross-section

- Squash ventral harder: `cy < 0` → multiply by **0.55** (was 0.75).
- Replace the single-gaussian `hump` with a piecewise arch: rise from
  t=0.08, peak **1.35×** at t=0.32, hold to t=0.55, drop. Currently 1.12×.
- Change defaults to a flattened shield: `mantleHeight = 0.22`,
  `mantleRadius = 0.42`.
- Add a **mantle collar** at `t ≈ 0.08`: insert a 2-segment reverse-taper
  where `profileScale` dips from ~0.85 to 0.70 before the head bulge.
  That's the mantle-opening lip visible in the refs.

### Fix 2 — `armSpine`: crown the arms, don't rake them

Replace `spreadY`/`spreadZ` with a **radial crown** around the mouth:

```
const theta = (idx / count) * Math.PI * 2;   // 0..2π around mouth
const crownR = 0.06;
start = new Vector3(mouthX,
                    mouthY + crownR * Math.sin(theta),
                    mouthZ + crownR * Math.cos(theta));
```

Then derive `forward/down/outward` from each arm's angular position so top
arms curl downward-forward and side arms curl inward-forward. Raise default
`armCurl` to **0.75** so tips converge at rest. Stagger pair length by
`[0.85, 0.95, 1.0, 1.1]` indexed by `local` (cuttlefish arm IV is longest).

### Fix 3 — `tentacleSpine` + `buildSideFin`: distinguish & thicken

**tentacleSpine:**

- Defaults: `tentacleLength = 1.8` (was 0.95), `tentacleBaseRadius = 0.018`
  (was 0.030), `tentacleClubRadius = 0.065` (was 0.040). 2× length, ~40%
  thinner, wider club.
- Move the `radialFn` bump inward: start at `t > 0.72` (was 0.82) so the
  club is a distinct swelling, not a bullet tip.
- Retracted routing: when `ext < 0.2`, p1/p2 must live **inside the arm
  crown** — pick `start.z` within `crownR`, not the current outside-of-arms
  placement (`0.045 + sideSign * 0.06 * (1-ext)`).

**buildSideFin:**

- `finWidth = 0.16` (was 0.10) and raise the `finMask` floor to 0.4 so the
  fin has presence end-to-end.
- Add **rest-curvature**: attach at `y = -p.mantleHeight * 0.15` (below
  dorsal midline) and curve the outer edge (`s = 1`) downward by
  `0.04 * profileScale` — gives the skirt shape from the side-view ref.
- Blend the first 20% of width (`s < 0.2`) into the mantle contour instead
  of stepping off it flat, to kill the hard-edge seam.

## Sheet-level note

The 4×4 needs **coupled deltas**, not single-axis sweeps:

- Rows should vary `(mantleLength, mantleRadius, mantleHeight, mantleTaper)`
  jointly — e.g. squat `(1.0, 0.48, 0.28, 0.35)` vs elongated
  `(2.2, 0.28, 0.18, 0.85)`. Currently only length moves meaningfully.
- Columns should vary `(armCurl, tentacleExtension, arm sway rotation)`
  jointly. Calm = `armCurl 0.9, tentExt 0.0, tentacles tucked`; striking =
  `armCurl 0.2, arms spread, tentExt 1.0`. Right now calm and drifting are
  visually identical.

Until the grid shows four genuinely different silhouettes across rows AND
four distinct postures across columns, the designer workflow isn't doing
its job.
